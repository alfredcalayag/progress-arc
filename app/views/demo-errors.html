DEMO Errors

Oh no! 'foo', 'bar', 'baz' were not valid data. We've broken the widgets... but wait... there's still a chance. They will still work if you revive them with valid inputs.

Valid numbers are floats from 0 to 1.0. Attempting a number greater will force to 1. Anything less than 0 or other will resort to 0.

Tips: Open the dev console and witness the helpful error messages that will save you hours in debugging.

Note: The inputs are controlled by the progress-arc widget itself. No special handling has been applied to the input fields on the parent controller.





1) input type="number" with strings in initial vm.data (i.e. "foo", "bar")
Actual:
Expected:
Reference Data: { actual: 'foo', expected: 'bar' }


2) input type="number" with out of bounds numbers in initial vm.data (i.e. -3, 100)
Actual:
Expected:
Reference Data: { actual: -3, expected: 100 }


3) input type="text" with number strings (i.e. "0.2" and "0.5" opposed to 0.2 and 0.5 respectively)
Actual:
Expected:
Reference Data: { actual: '0.2', expected: '0.5' }


4) input type="text" with string characters (i.e. "foo", "bar")
Actual:
Expected:
Reference Data: { actual: 'foo', expected: 'bar' }
